Start with Keras

https://keras.io/zh/ 從這裡開始。

要在後端安裝tensor flow，Keras只是一個API。

https://www.tensorflow.org/install/

pip install --upgrade pip: 先把pip升級到最新版。

成功把pip從 20.1.1升級到20.3。

pip install tensorflow接著下載tensor flow。.whl 文件165.2MB，緊接著報錯。

在stake overflow上找到一個相似的問題，https://stackoverflow.com/questions/62241930/error-exception-traceback-most-recent-call-last-tensorflow 。在CSDN上找到一個更靠譜的回答： https://blog.csdn.net/happy1223247773/article/details/105697268 。決定把科學上網換全局模式再試一遍。再不行可能就考慮換清華的源。事實證明正常的下載速度大約是1M/S。

接著試試看能不能安裝Keras了。

sudo pip install keras 出現了warning，但是顯示了Successfully installed keras-2.4.3.

有Keras就一定會提到GPU，下次在教學服務器上配環境時可能需要。在此之前先擱置掉它。


First Time for Keras and NN

之前下載好了Conda和PyCharm，下面嘗試找一個CIFAR-10的項目照搬下來。

嘗試這個“教程”： https://towardsdatascience.com/deep-learning-with-cifar-10-image-classification-64ab92110d79

出現第一個問題：PyCharm上import tensorflow does not work.

決定從這篇文章入手，https://databricks.com/blog/2018/07/10/how-to-use-mlflow-tensorflow-and-keras-with-pycharm.html 。

然後自己在PyCharm裡到處找setting to install tensor flow，遇到了這樣的問題：

  • tensorflow -> python[version=‘2.7.|3.7.|3.6.|3.5.’]

Your python: python=3.8

If python is on the left-most side of the chain, that’s the version you’ve asked for.
When python appears to the right, that indicates that the thing on the left is somehow not available for the python version you are constrained to. Note that conda will not change your python version to a different minor version unless you explicitly specify that.


然後重新用PyCharm提供的虛擬環境安裝，出現了同樣的錯誤。“Try to run this command from the system terminal. Make sure that you use the correct version of ‘pip’ installed for your Python interpreter … ”
