Context

經過了前面兩天的摸魚生活，我已經把計網的內容忘得差不多了：罪過罪過。昨天把微博備份下來後，本打算放在網頁上，但是最後只是個失敗了的嘗試。

Plan

說是plan，實際上很多都是在思考“雖然我沒有ADHD，但是那個channel裡的視頻對我來說好像很有用的樣子”。而且我發現我確實在一天之內如果完成大部頭的工作、還無跡可尋的時候，就非常容易摸魚。閒話少說：比如高中時能夠按部就班地學，也是因為有教輔資料、教輔資料不會讓人覺得某一部分難度大如劈叉。如果需要大跨度如劈叉，對我來說“存檔”是有必要的。

當然現在第一步是把YouTube網頁關掉，然後整理學習資料。計畫是整理完成之後按照review提到的內容自己複習。在此之前，還需要一個表，列出自己可能遇見的困難。至於解決方法？困難都是小困難，只是到了我這兒就會阻礙我複習罷了。

Conception Interpretation
Thumb code 相對於32位精簡指令集而言，只有16位。能作為對ARM的補充但不能單獨使用。
barrel shifter 是一種“功能單元”，提供shift功能的。（猜測：對於這種功能單元，應該有自己的指令。如果有，是不是需要記住？）在一個網站上看到了估計對硬件指令比較全的介紹。
Booth Algorithm 一種在二進制下算乘法很快的辦法。把“0 / 1” 視作是否右移。(❗️但是不知道怎麼算啊。 )
Mnemonics 中文翻譯是助記符。（在這裏提到了r15: hold the memory address of the next instruction to be executed.）
Mnemonic and Instructions ADD rz, ry, rx:add the value in register x to the value in register y and place the sum in register z.
The same with AND and OR.
Flag ARM condition flag，可能有zero flag / Negative flag 等等。Flags are essentially 1 bit memory devices and different microprocessors have different flags. Carry flag: 數據作為int類型有溢出（232−12^{32} - 1232−1）。
Representation？ 指的是對character的表示、code的方式。（比如ascii碼）
還有對於負數的表達方式：一的補碼和二的補碼？二的補碼是全部顛倒後再+1+1+1的。
還有對浮點數的表示方式，標準是IEEE。
link register 當一個subroutine結束之後要告訴程序回到哪一條指令。ARME裡面的r14(用lr助記符往裡面寫)是用來寫main program的位置的。
BL助記符用來寫“branch and link”。在subroutine結束時用MOV pc, lr.來回到主程序。
interrupts 主程序中某一個部分出現意外時，執行interrupt程序，完成之後再回到主程序繼續執行原定代碼。
ARM7有兩種中斷：IRQ / FIQ？？
IRQ / FIQ FIQ的優先度比IRQ要高。IQs for fast, low latency interrupt handling and Interrupt Requests (IRQs), for more general interrupts.
對於IRQ：ox0000 0018，對於FIQ：0x0000 001C用來存主程序被打斷的地方，也是interrupt vector。
ADDNE 看到了另一個解釋，Usual ADD doesn’t update flags. ADDS does.
看課件（P53）的意思是，如果r11目的寄存器的zero flag是空的話，就往r11裡面存。
Load / Store Load是把數據從memory往register傳。
Store 是把數據從register裡存到memory中。
register只存4個bytes，但是memory可能是4個或者8個bits。
LDR&STR: 是ARM的指令。一個是load register, 把data從memory中取出來，允許有偏移量，一次性取出、不只是一個word。STR是把data放進memory中，記得偏移量×4\times 4×4。
little endian 大小端的問題。小端的意思是越往右越重要（因為通常是4個byte連在一起）。如果問"first or last more significant"，last指靠右，first靠左，小端 last byte most significant. 日常十進制數字是大端表示的。
Tri-state gates 三態門？有三態：0 / 1 / 高阻態。OE是使能信號，如果OE為0，輸出是高阻態，與IN的輸入無關。
processor mode 和不同的操作系統有關係，會決定如何讀取、理解memory上的數據。
