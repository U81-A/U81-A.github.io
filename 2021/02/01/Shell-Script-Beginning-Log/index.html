How to Start

最近的狀態就是摸魚、摸魚、摸魚。

看了下，兩個OS的作業都是明天下午6點鐘截止。於是開了虛擬機開始升級ubuntu然後又把它停下了，然後又升級了。似乎body double對我來說有用。anyway，先看一遍Lab tutorial。

感覺今天的網速很不錯，說不定我今天上午能升級成功。然後開始看tutorial，有13\frac{1}{3}31​都是在講VIM：還需要重新學習。


Start VIM

這篇https://www.maketecheasier.com/cheatsheet/vim-keyboard-shortcuts/ 的感覺不錯（至少能看懂）。按照教程，在terminal裡面輸入vimtutor會自動彈出教程。（ubuntu升級的過程：告訴我下載新文件需要56min，升級需要數小時。）這樣看來我還是現在macOS上用terminal看著吧。

雖然有一些“翻譯腔”的痕跡，但是我還是忍不住說“太美麗了”。


本來想看一下vimtutor有多長，然後搜到了stackoverflow上的這篇問題，https://superuser.com/questions/246487/how-to-use-vimtutor。似乎在這上面的問題也是我一開始可能會碰到的問題，只是運氣好所以直接進入了。

雖然說教程說可能需要25～30分鐘，但是我總是做好要用一個小時的打算了。於是我先去Vivaldi上面看一下那篇提到vim tutor的教程的網頁：是在11:20打開的。現在12:20，那可能我還需要半個小時？


OS_T00_C.pptx

按照教程在ubuntu的shell上寫了第一個C語言的代碼。似乎還比較順利，另外記得老師的意思好像是這些過程會要求截圖提交，準備把PPT看完後再統一截圖。（snipaste真好用）


Start the Test Report 1

根據網站上的意思，該測試“只能進行一次、可保存並稍後繼續”。於是我就直接點開了。希望今天晚上能做完其中一個？另外一個或許有點懸。看來睡眠時間會被壓榨了。anyway，都是我自找的。然後我發現是另一個教程的實現過程需要截圖，希望今天能把report寫完。
