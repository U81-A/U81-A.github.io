Start of Using GitHub Repository as Local's Backup

Posted on 2021-02-18 In Tech
Symbols count in article: 1k

Context

在整理自己電腦上的大文件時，有一個網易雲音樂的下載文件夾，大約1～2個G，於是準備把整個文件夾裡的內容備份到GitHub上。


Code Part

  • 如果想刪掉git有關的所有內容： rm -rf .git。原理：在git init文件夾下會有一個隱藏文件夾叫git，因為是隱藏的所以是“.git”。

  • 如果先在GitHub上新建了遠程倉庫：如https://github.com/U81-A/NeteaseMusic。則先要把遠程倉庫clone到本地，此時本地會有一個和遠程倉庫同名的文件夾，需要備份到的文件會先移到此文件夾中，然後commit、push。


  1. git init:new a git
  2. git clone https://github.com/U81-A/NeteaseMusic: clone the remote repository to local, and create a folder with same name with repository.
  3. cd .NeteaseMusic: after copy all the file un-backuped into the newly created folder, step into it.
  4. git add *: add all the files (all the mp3 file in my terms of operation) to git, let git track them.
  5. git commit -m "first try": commit all the songs to git. You can add several times before a final commit.
  6. git status & git branch --unset-upstream: check the status. If there’s some problem, do the command git asks.
  7. git push -u origin master: finally push the committed songs to remote repository.

Caution

  • The first time you do git add * may take a little longer, but try not stop it or suspend it. I tried and remove all the git thing to restart.
  • remember to clone the remote repository before do git add or commit command, otherwise the branch may be a total mess.
